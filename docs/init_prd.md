# PRD v1.0

## 项目名称

**认知盲区拼图 · Blindspot Puzzle**

## 一句话定位

**在知乎指定圈子的真实讨论里，识别“谁缺什么、谁会什么”，让 Second Me 分身先完成互补分析与协作撮合，再决定真人是否连接。**

---

# 1. 项目背景

当前很多 A2A 项目都在做“同频”“共鸣”“人格相似”匹配，但现实里更高价值的连接，往往不是“你和我很像”，而是：

* 你擅长的，正好是我不会的
* 我能提供的，正好补上你的短板
* 双方不需要先寒暄，也能快速确认是否值得合作

“认知盲区拼图”想解决的不是“要不要认识”，而是：

**能不能在真实问题场景里，快速找到一个能力互补的人。**

Second Me 提供用户身份、分身和 A2A 对话能力；知乎提供真实社区讨论语境。两者结合后，可以把“讨论关系”进一步升级为“协作关系”。开发者手册明确要求项目接入 Second Me OAuth，且必须是 A2A 应用或面向 Agent 的应用，Agent 必须是核心参与者。

# 2. 产品目标

## 2.1 产品目标

构建一个基于 Second Me 分身的**互补协作发现工具**，帮助用户在知乎指定圈子的真实内容中发现潜在合作对象，并由 Agent 先输出一份“合作提案”。

## 2.2 核心价值

1. 降低合作连接的社交成本
2. 将“讨论内容”转化为“协作机会”
3. 让 Agent 不只是聊天，而是代替用户做互补识别与初步协商
4. 与“同频社交”区分，聚焦“异质互补”

## 2.3 黑客松价值

这个项目天然符合：

* Second Me 的 A2A 场景要求
* 知乎“人类社区重新定义连接”的主题
* 知乎特别奖对社区能力调用的期待

# 3. 产品定位

## 3.1 不做什么

本项目**不做**：

* 全网开放式找人平台
* 基于知乎私域历史行为的画像抓取
* 热点问答人格测试
* 只输出“第一印象报告”的社交匹配器

## 3.2 做什么

本项目要做的是：

**在指定圈子的真实讨论上下文中，发现互补关系，并输出合作提案。**

即：

* 不是找“和你一样的人”
* 而是找“刚好补你盲区的人”

# 4. 用户问题

用户当前面临的主要问题：

1. 在公开讨论里，用户经常暴露出“问题”和“盲区”，但没人把它转化成合作机会
2. 用户可能很有能力，但不擅长主动开口寻求互助
3. 即使发现潜在对象，也不知道怎么自然发起连接
4. 传统社交产品擅长找兴趣相近者，不擅长找能力互补者
5. 合作前需要试探、寒暄、解释价值，成本很高

## 5.1 核心用户

* 有明确专业长板的人
* 有现实问题待解决的人
* 不擅长主动社交但愿意合作的人
* 希望借助 Agent 完成前置沟通的人

## 5.2 典型用户举例

* 运营人：能提供增长建议，但缺产品技术落地能力
* 创作者：能提供内容创作，但缺商业合作或技术支持

# 8. 需求描述

“认知盲区拼图”通过读取指定知乎圈子的帖子、评论与互动内容，识别讨论中暴露出的两类信号：

* **能力长板**：某用户在某类问题下展现出明显的理解力、判断力、经验或输出能力
* **认知短板**：某用户在某类问题下暴露出明显的需求、困惑、知识空缺或求助意图

系统不追求“观点相似度”，而是追求“能力互补度”。
当系统识别到两个用户在同一讨论语境下存在“你会的正好补我不会”的关系时，将触发两个 Second Me Agent 的互补分析流程，并生成一份结构化合作提案。

真人用户只需要在最后确认：

* 要不要认识对方
* 要不要接受这次合作提案

---

# 9. 产品概述

## 9.1 产品核心链路

**圈子内容 → 识别长板/短板 → 发现互补对象 → Agent 双向分析 → 输出合作提案 → 双方确认后连接**

## 9.2 差异化

和“同频雷达”不同，本产品不是：

* 根据几道题看两个人像不像
* 输出第一印象或共鸣度

而是：

* 在真实讨论里看谁能补谁
* 输出一份“值不值得合作”的提案

## 9.3 结果导向

产品输出的是：

* 互补说明
* 协作建议
* 初步交换方案

而不是仅仅“想不想认识”。

# 11. 用户旅程

## 旅程 A：从圈子内容发现互补对象

1. 用户用 Second Me 登录
2. 进入指定知乎圈子
3. 浏览帖子和评论
4. 点击“发现互补机会”
5. 系统分析该讨论中涉及的能力点和需求点
6. 找到潜在互补对象
7. 双方 Agent 生成互补分析报告
8. 输出合作提案
9. 双方确认后再建立进一步连接

## 旅程 B：从“求补位”发帖触发协作

1. 用户用 Second Me 登录
2. 填写自己当前的难题和可交换技能
3. Agent 生成一条“求补位”帖子
4. 帖子发到指定知乎圈子
5. 系统跟踪评论与互动
6. 识别潜在互补回复者
7. Agent 发起双向互补分析
8. 输出合作提案
9. 双方确认后连接

# 13. 实现逻辑

## 13.1 身份接入逻辑

* 用户通过 Second Me OAuth 登录
* 系统获取用户 Second Me ID 和基础信息
* 系统绑定该用户分身
* 后续所有 Agent 分析和协作建议，都以此身份为基础

## 13.2 圈子内容接入逻辑

* 读取两个指定圈子的帖子内容和评论
* 将帖子、评论、点赞等互动作为“互补信号池”
* 不做全站内容抓取
* 不做用户私域历史数据接入

## 13.3 长板/短板识别逻辑

系统基于以下信号判断：

* 帖子内容是否在提供经验、方案、判断
* 评论是否在求解、追问、表达卡点
* 是否存在持续高质量回应
* 是否出现高密度专业词汇和结构化表达
* 是否出现明确的资源交换意图

输出：

* 长板标签
* 短板标签
* 协作场景标签
* 置信度

## 13.4 互补匹配逻辑

系统不是算“相似度”，而是算：

* 我的长板是否补你的短板
* 你的长板是否补我的短板
* 双方是否在同一讨论语境下出现
* 双方是否有共同问题背景
* 双方是否存在现实可执行的合作形式

## 13.5 Agent 协作分析逻辑

当识别到一组高互补对象时：

* 用户 A 的 Agent 总结：我能为对方做什么
* 用户 B 的 Agent 总结：我能为对方做什么
* 两个 Agent 分别形成判断
* 系统融合输出一份合作提案

## 13.6 热榜增强逻辑

热榜用于：

* 给当前圈子讨论补充更大的社会议题背景
* 帮助 Agent 理解“现在大家在聊什么”
* 让发帖或提案更贴近中文互联网语境

## 13.7 可信搜增强逻辑

可信搜用于：

* 在生成提案前检索相关讨论
* 提高协作建议的内容依据
* 使用 `authority_level` 过滤更高质量内容
  但因调用量有限，必须做缓存，不可作为高频实时依赖。

# 14. 详细功能描述

## 14.1 功能一：Second Me 登录

**输入**：用户点击登录
**输出**：用户身份、Second Me ID、分身上下文
**成功标准**：可以进入应用主页并发起分身对话

## 14.2 功能二：互补画像生成

**输入**：

* 用户填写内容
* 分身盘点对话
* 圈子内容线索

**输出**：

* 我的长板
* 我的短板
* 我能提供的帮助
* 我想获得的帮助

## 14.3 功能三：圈子内容理解

**输入**：圈子帖子、评论、互动数据
**输出**：

* 话题分类
* 需求点
* 能力点
* 潜在互补对象列表

## 14.4 功能四：互补分析

**输入**：两名用户在同一讨论中的表达
**输出**：

* 互补理由
* 共同上下文
* 协作场景建议
* 交换价值说明

## 14.5 功能五：Agent 合作提案

**输入**：互补结果 + 双方边界
**输出**：

* 本次合作主题
* 我提供什么
* 对方提供什么
* 推荐协作方式
* 真人确认入口

## 14.6 功能六：知乎圈子发帖

**输入**：标题、正文、图片、圈子 ID
**输出**：发帖成功并返回 content\_token

## 14.7 功能七：评论与点赞

**输入**：目标内容 token / comment id
**输出**：互动结果
**限制**：仅对白名单圈子内容生效

# 17. 异常场景

1. **OAuth 失败**
   提示重新授权

2. **圈子接口调用失败**
   展示降级页并允许稍后重试

3. **内容不属于白名单圈子**
   不允许点赞或评论

4. **发帖字段缺失**
   返回失败，例如 title is required

5. **可信搜超限**
   使用缓存结果或跳过检索增强

6. **接口限流**
   命中 10 QPS 限制时，进入排队与重试

# 20. 风险与对策

## 风险 1：和“同频匹配”项目混淆

**对策**：始终强调“互补”“补位”“协作提案”，不强调“同频”“共鸣”

## 风险 2：知乎能力边界被误写

**对策**：明确只支持两个圈子，不描述全站找人

## 风险 3：A2A 不够强

**对策**：让两个用户 Agent 都参与互补分析，避免退化为单 Agent 工具

## 风险 4：可信搜被高频打爆

**对策**：只在关键节点调用，并强制缓存

# 21. 最终项目简介（可直接用）

**认知盲区拼图** 是一个基于 Second Me 分身与知乎指定圈子内容的互补协作发现工具。
它不寻找“和你一样的人”，而是在真实讨论语境中识别“谁刚好能补上你的盲区”。
