## Context

当前实现：
- Step 2 显示 AI 匹配结果（matches 数组）
- 进入 Step 3 协商时，代码固定使用 `matches[0]`：`const targetPost = matches[0].post`
- 用户无法主动选择

## Goals / Non-Goals

**Goals:**
- 用户可以在 Step 2 匹配结果中选择一个帖子
- 选中的帖子在 Step 3 协商中使用
- 选中状态有明确的视觉反馈

**Non-Goals:**
- 不改变协商 API 的数据格式
- 不添加多选功能（单选）
- 不修改 Step 1 帖子列表的交互

## Decisions

### Decision 1: 选择机制使用单选卡片模式

**选择：** 点击匹配卡片切换选中状态，选中的卡片显示高亮边框

**UI 示意：**
```
┌─────────────────────────────────────────┐
│  ┌─────────────────────────────────┐   │
│  │ ○ 张三的讨论...  匹配度 85%     │ ← 未选中
│  └─────────────────────────────────┘   │
│  ┌─────────────────────────────────┐   │
│  │ ● 李四的讨论...  匹配度 72%     │ ← 选中（蓝色边框）
│  │   ✓ 已选择                        │   │
│  └─────────────────────────────────┘   │
│                                         │
│  [ 发起 Agent 协商 → ]                 │
└─────────────────────────────────────────┘
```

**替代方案：**
- 复选框列表 → 适合多选，但不需要多选
- 下拉选择 → 需要额外点击，不如卡片直观

### Decision 2: 默认选中第一个

**选择：** 如果用户没有手动选择，进入协商前自动选中 `matches[0]`

**理由：** 保持向后兼容，用户体验更流畅

## Risks / Trade-offs

| Risk | Mitigation |
|------|------------|
| 用户忘记选择直接点协商 | 默认选中第一个，不影响流程 |
| 选中后返回 Step 2 选择其他 | 保持选中状态直到明确改变 |
