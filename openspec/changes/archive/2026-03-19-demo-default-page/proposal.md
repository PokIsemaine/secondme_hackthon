# Proposal: Demo Default Page

## 1. Summary

将 `/demo` 设为默认入口页面，将原首页功能融合到 demo 页面，删除独立的协商室页面，保持所有功能在单页面内完成。

## 2. Problem Statement

当前应用存在以下问题：
1. 存在多个独立页面（首页、demo 页、协商室、发帖页），不符合 PRD v2 要求的"单页面用户旅程"
2. 用户需要在多个页面间跳转，体验碎片化
3. `/demo` 页面已实现完整功能链路，但非默认入口

## 3. Proposed Solution

### 3.1 路由调整
- 将 `/demo` 设为默认入口（将 `src/app/page.tsx` 重定向到 `/demo`）
- 删除 `src/app/negotiation/` 和 `src/app/negotiation/[id]/` 路由
- 保持 `src/app/seeker/` 路由（求补位发帖功能可作为独立入口）

### 3.2 Demo 页面增强
将原首页功能融合到 demo 页面：
- 登录按钮和用户信息展示
- 用户画像展示/编辑
- 圈子内容发现
- AI 匹配入口
- 协商过程展示
- 提案结果展示

### 3.3 页面流程（单页面）
```
进入 → 检查登录状态
       ↓
  未登录 → 显示登录入口 + 引导
       ↓
  已登录 → 进入主流程
       ↓
  步骤1：我的画像 → 步骤2：发现内容 → 步骤3：AI匹配 → 步骤4：协商过程 → 步骤5：提案结果
```

## 4. Scope

### 4.1 必做
- [ ] `/` 重定向到 `/demo`
- [ ] 将登录检测和用户信息展示融入 demo 页面
- [ ] 将用户画像组件融入 demo 页面
- [ ] 将圈子发现组件融入 demo 页面
- [ ] 删除 `/negotiation` 和 `/negotiation/[id]` 路由

### 4.2 不做
- 复杂的路由结构
- 独立的多页面体验

## 5. Success Criteria

- 用户访问 `/` 自动跳转到 `/demo`
- 用户可在单一页面完成完整的"发现 → 协商 → 提案"流程
- 不存在独立的协商室页面
