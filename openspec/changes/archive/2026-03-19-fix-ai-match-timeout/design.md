## Context

当前 `/api/circle/match` API 调用 SecondMe API 时没有设置超时限制，可能导致前端长时间等待无响应。

## Goals / Non-Goals

**Goals:**
- 添加 fetch 请求超时机制
- 返回明确的超时错误信息
- 改善用户体验

**Non-Goals:**
- 不改变 API 响应格式
- 不修改匹配逻辑

## Decisions

### D1: 超时策略
**选择**: 使用 AbortController 设置 30 秒超时

**理由**:
- 30 秒对于 LLM API 调用是合理的等待时间
- AbortController 是原生 API，无需额外依赖
- 可以区分超时和其他错误

### D2: 错误处理
**选择**: 超时时返回特定错误码，方便前端区分处理

**理由**:
- 前端可以根据错误码显示不同提示
- 便于调试和问题定位

## Risks / Trade-offs

- **风险**: 30 秒超时可能对复杂查询不够
  - **缓解**: 可配置超时时间，当前设置为合理默认值

- **风险**: 快速失败可能影响成功率
  - **缓解**: 保留重试机制在前端处理
